# Geoffrey Moreno — Portfolio Project Rules

## Project Context
This is a professional portfolio for Geoffrey Moreno, targeting full-time Creative Strategist / Project Manager roles.
The portfolio showcases short-form content strategy and execution for multiple brands across different sectors.
Main formats: TikTok, Reels, Shorts. Content types: organic, paid, and influencer campaigns.
Stack: HTML, CSS, JavaScript only. No frameworks.

## Visual Identity — Never Override
- Use Poppins only.
- Headings: ExtraBold / Bold.
- Body text: Medium.
- Dark mode only. Never add light mode or a theme toggle.
- Main background: dark navy textured atmosphere.
- Accent color: #E50914 or brand-adapted equivalent for project pages.
- Primary text: soft white, high readability without harsh brightness.
- Never use text-align: justify.

## Layout Behavior
- When moving an element, never move it in isolation.
- Rebalance surrounding spacing, alignment, and proportions automatically.
- If the request is vague ("more to the right", "slightly bigger"), choose the most visually coherent solution based on nearby elements and explain the choice briefly.
- Prefer flex, grid, gap, margin. Avoid unnecessary absolute positioning.

## Typography Rules
- H1: Poppins ExtraBold, uppercase, strong presence.
- H2: Poppins Bold, uppercase.
- H3: Poppins SemiBold.
- Body: Poppins Medium, left-aligned.
- Typography must stay coherent across all sections and pages.

## Content Blocks
- Main page structure: Navbar → Hero → Brands / trust logos → Projects grid → Contact.
- Project page structure: Hero → Context / Missions → Results / Videos / Stats.
- Keep a premium, structured, editorial feel.
- If a section feels empty or visually weak, suggest a better content distribution.

## Brand Adaptation
- Project pages may adapt colors to each brand while keeping the same global presentation logic.
- Only adapt color variables and subtle visual accents unless explicitly asked to change structure.
- Confirm HEX colors before applying a new brand theme when needed.

## Animations
- Preserve working animations unless explicitly asked to modify them.
- When integrating animation code provided by the user, keep its logic intact.
- Suggest subtle premium motion only when it improves hierarchy or interaction clarity.

## Debug Rules
- Diagnose before fixing.
- Identify the likely root cause in one sentence.
- Apply surgical fixes only.
- Never rewrite a whole section to fix a local bug.

## Force of Proposal
After each modification:
- Suggest one relevant visual or UX improvement.
- Flag any inconsistency in spacing, hierarchy, typography, color, or interaction.

## Absolute Prohibitions
- Never change font family.
- Never delete existing sections unless explicitly requested.
- Never add light mode.
- Never use justified body text.
- Never invent a totally new project layout if a template or existing logic is already in place.
